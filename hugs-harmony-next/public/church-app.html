ตั้งค่า API / Webhook (Real Connection)

เมื่อกำหนด URL ไว้ ระบบจะส่ง (POST) ข้อมูล JSON ออกไปยังปลายทางนี้แบบทางเดียวทุกครั้งที่มีการบันทึกรายรับ/รายจ่าย/อนุมัติใบเสร็จ/ปิดรอบสัปดาห์ เพื่อให้คุณนำไปต่อยอดเก็บลง Google Sheets หรือระบบอื่นได้ (เว้นว่างได้หากยังไม่ต้องการเชื่อมต่อ)

การจัดการสำรองข้อมูล (Database JSON)

ดึงข้อมูลทั้งหมดออกเป็นรูปแบบไฟล์ดิบ (JSON Backup) เพื่อใช้สำหรับงานพัฒนาระบบคลาวด์หรือดึงข้อมูลคืนสภาพ (Restore)

📘 หมายเหตุสำหรับนักพัฒนา:

Webhook ของระบบนี้เป็นการส่งออกทางเดียว (fire-and-forget) เหมาะกับ Google Apps Script / Make.com ที่รับ JSON แล้วบันทึกลง Sheet ได้ทันที ระบบไม่รองรับการรับข้อมูลย้อนกลับเข้ามาอัตโนมัติ — ทุกรายการยังต้องกรอก/ตรวจสอบในแอปนี้เสมอ

คำถามที่พบบ่อย

บันทึกรายรับ-รายจ่ายยังไง?

ไปที่ "บันทึกรายรับประจำสัปดาห์" กรอกยอดถวายแยกตามประเภทของแต่ละคน หรือไปที่ "บันทึกใบเบิกจ่ายเงิน" สำหรับรายจ่าย กดบันทึกแต่ละรายการได้ทันที ไม่ต้องรอปิดสัปดาห์

แนบใบเสร็จทำงานอย่างไร?

ไปที่ "แนบใบเสร็จ / รอตรวจสอบ" แนบรูปใบเสร็จจริงพร้อมกรอกรายละเอียดผู้เบิกและจำนวนเงินด้วยตนเอง (ระบบไม่มีการอ่านข้อมูลอัตโนมัติ) รายการจะเข้าคิวรอ แล้วต้องกด "อนุมัติ" อีกครั้งก่อนจะตัดเข้าบัญชีจริง

ปิดยอดวันอาทิตย์ทำไปทำไม?

เพื่อกระทบยอดเงินสด/เงินในบัญชีจริงกับตัวเลขในสมุดบัญชี ก่อนยกยอดคงเหลือไปเป็นยอดยกมาของสัปดาห์ถัดไป และล้างตารางรายรับ-รายจ่ายให้เริ่มสัปดาห์ใหม่

ตั้งค่า Webhook เพื่ออะไร?

ถ้าใส่ URL ปลายทางไว้ในหน้าตั้งค่า ระบบจะส่งข้อมูลออกไปอัตโนมัติทุกครั้งที่บันทึกรายรับ/รายจ่าย/อนุมัติใบเสร็จ/ปิดสัปดาห์ เพื่อให้เชื่อมกับ Google Sheets หรือระบบอื่นที่คุณตั้งไว้ (ไม่บังคับ)

ข้อมูลเก็บไว้ที่ไหน ปลอดภัยไหม?

ข้อมูลทั้งหมดเก็บในเบราว์เซอร์ของเครื่องที่เปิดแอปนี้เท่านั้น (ไม่ sync ข้ามเครื่อง) ควรกด "Backup (JSON)" ในหน้าตั้งค่าเป็นประจำเพื่อสำรองข้อมูลไว้กันเครื่องหาย
